# Architecture Decision Records

This directory stores Architecture Decision Records (ADRs) for the project.

## ADR workflow

1. Read existing ADRs before planning any implementation task.
2. Create a new ADR when a refactor introduces or changes architecture decisions.
3. Keep ADR status explicit (`proposed`, `accepted`, `superseded`, `deprecated`).
4. Link relevant ADRs from the README and implementation plan.
